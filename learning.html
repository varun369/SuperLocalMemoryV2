Layer Architecture
Three Learning Layers

Each layer captures a different dimension of your behavior. Together they build a complete model of how you work, entirely on your local machine.

01
Tech Preferences
Detects your preferred technologies, frameworks, and patterns across all projects. Transfers knowledge between workspaces.
Python FastAPI pytest TypeScript SQLite Docker Tailwind Astro
Click to expand
The Cross-Project Aggregator scans all your memories to detect technology preferences. It learns that you prefer pytest over unittest, FastAPI over Flask, TypeScript over plain JavaScript. These preferences transfer between projects automatically, so when you start a new project, your AI already knows your stack.
02
Project Context
4-signal detection identifies which project you are working in and boosts relevant memories automatically.
Tags File Paths Profile Graph Clusters
Click to expand
The Project Context Manager uses four signals to detect your current project: memory tags, file path patterns in recent memories, active profile, and knowledge graph cluster proximity. When you are working on "SuperLocalMemory", it boosts SLM-related memories. Switch to a different project, and context automatically shifts.
03
Workflow Patterns
Learns your temporal work sequences using sliding-window mining. Predicts what you will need next.
docs → arch → code → test → deploy
Click to expand
The Workflow Pattern Miner uses sliding-window sequence analysis (based on PrefixSpan) to discover your temporal work patterns. If you typically read docs, then design architecture, then write code, then write tests, the system learns this sequence and proactively surfaces the right memories at the right time in your workflow.
Ranking Evolution
Three-Phase Adaptive Ranking

The ranking system automatically evolves from simple to sophisticated as you use it. No configuration needed. The system bootstraps ML features from your existing data on day one.

Phase 0
Baseline
0 learning signals
Standard FTS5 + TF-IDF retrieval. No personalization. Same as v2.6 and earlier.
Phase 1
Rule-Based
20+ signals
Hand-crafted boosting rules: recency, frequency, project match, tech preference overlap, source quality.
Phase 2
ML Model
200+ signals
LightGBM LambdaRank with synthetic bootstrap. Full ML ranking from day 1. Improves with every interaction.
Install First feedback 10+ sessions Fully personalized
Privacy First
Privacy by Design

Learning about you does not mean sending your data anywhere. Every byte stays on your machine. Compare with cloud alternatives.

Cloud Alternatives
  • Your data stored on their servers
  • Their terms of service, their control
  • Submit deletion request, wait weeks
  • Usage analytics tracking your behavior
  • $24-50/month subscription fees
  • Vendor lock-in to proprietary formats
  • Data may be used for model training
SuperLocalMemory
  • Your data on YOUR machine only
  • Two SQLite files you fully own
  • slm learning reset -- instant delete
  • Zero telemetry. Absolutely none.
  • Free forever. No API keys needed.
  • Open SQLite format, MIT licensed
  • Your data never leaves localhost

Get Started

Install SuperLocalMemory in under 60 seconds. The learning system activates automatically in v2.7.

# Install or upgrade
npm install -g superlocalmemoryv2@latest

# Check learning status
slm learning status

# Reset learned data (GDPR Art. 17)
slm learning reset
